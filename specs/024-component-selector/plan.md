# Implementation Plan: Multi-Component Evolution with Component Selectors

**Branch**: `024-component-selector` | **Date**: 2026-01-14 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/024-component-selector/spec.md`

## Summary

Add component selector strategies (round-robin, all-at-once) to enable multi-component evolution in gepa-adk. This feature replaces the hardcoded `components_to_update = ["instruction"]` in `AsyncGEPAEngine._propose_mutation()` with configurable selector strategies that cycle through or select all candidate components, enabling schema evolution and multi-agent workflow improvement.

**Technical Approach**: Define `ComponentSelectorProtocol` in ports layer, implement `RoundRobinComponentSelector` and `AllComponentSelector` adapters mirroring upstream GEPA patterns, inject selector into engine, and expose via `component_selector` parameter in public API functions.

## Implementation Phases

### Phase 1: Protocol & Port Layer

1. Extend `src/gepa_adk/ports/selector.py`:
   - Add `ComponentSelectorProtocol` with `select_components()` async method
   - Export from `ports/__init__.py`

### Phase 2: Adapter Implementations

1. Create `src/gepa_adk/adapters/component_selector.py`:
   - Implement `RoundRobinComponentSelector` with per-candidate-idx state tracking
   - Implement `AllComponentSelector` (stateless)
   - Add `create_component_selector()` factory function
   - Export from `adapters/__init__.py`

### Phase 3: Engine Integration

1. Modify `src/gepa_adk/engine/async_engine.py`:
   - Add `component_selector` parameter to `__init__`
   - Replace hardcoded `components_to_update = ["instruction"]` in `_propose_mutation()`
   - Build component list from candidate keys
   - Call selector to get components_to_update

### Phase 4: API Integration

1. Modify `src/gepa_adk/api.py`:
   - Add `component_selector` parameter to `evolve()`, `evolve_group()`, `evolve_workflow()`
   - Resolve string to selector instance using factory
   - Pass to engine constructor

### Phase 5: Testing

1. Add contract tests: `tests/contracts/test_component_selector_protocol.py`
2. Add unit tests: `tests/unit/adapters/test_component_selector.py`
3. Add engine unit tests: `tests/unit/engine/test_engine_component_selection.py`
4. Add integration tests: `tests/integration/test_multi_component_evolution.py`
