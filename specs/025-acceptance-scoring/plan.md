# Implementation Plan: Align Acceptance Scoring with Upstream GEPA

**Branch**: `025-acceptance-scoring` | **Date**: 2026-01-15 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/025-acceptance-scoring/spec.md`

## Summary

Align acceptance decisions with upstream GEPA by using sum-based aggregation for
iteration evaluation batches while keeping mean-based valset tracking. Add an
acceptance scoring mode flag for backward compatibility and expose it via
EvolutionConfig in the public API.

**Technical Approach**: Introduce `acceptance_metric` in EvolutionConfig, compute
acceptance scores via sum or mean depending on the configured mode, keep valset
tracking as mean, and store acceptance vs valset scores separately in the engine
state/result when needed. Tests cover contract expectations, unit acceptance logic,
and integration behavior.

## Implementation Phases

### Phase 0: Research (Complete)

- Confirm upstream GEPA acceptance aggregation uses sum of subsample scores in
  `.venv/lib/python3.12/site-packages/gepa/core/engine.py`.
- Document decisions in `research.md`.

### Phase 1: Domain + API Configuration

1. Update `src/gepa_adk/domain/models.py`:
   - Add `acceptance_metric: Literal["sum", "mean"] = "sum"` to EvolutionConfig
   - Validate allowed values and raise ConfigurationError on invalid input

2. Update `src/gepa_adk/api.py`:
   - Accept and pass through acceptance_metric via EvolutionConfig
   - Ensure valset reporting remains mean-based

### Phase 2: Engine Acceptance Semantics

1. Update `src/gepa_adk/engine/async_engine.py`:
   - Compute acceptance score based on acceptance_metric
   - Keep valset reporting as mean
   - Store acceptance and valset scores distinctly if required for reporting

2. Update `src/gepa_adk/domain/exceptions.py`:
   - Add/extend errors for empty or non-finite score lists (if not already covered)

### Phase 3: Testing (Three-Layer)

1. Contract tests:
   - `tests/contracts/test_acceptance_scoring_contract.py` for config validation
2. Unit tests:
   - `tests/unit/engine/test_acceptance_scoring.py` for sum vs mean acceptance
3. Integration tests:
   - `tests/integration/test_acceptance_scoring.py` for end-to-end behavior
